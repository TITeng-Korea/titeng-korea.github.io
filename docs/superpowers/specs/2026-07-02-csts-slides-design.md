# CSTS HTML Slides Design

**Date:** 2026-07-02

**Goal**

`posts/csts/csts.md`의 전체 내용을 빠짐없이 포함한 16:9 비율의 강의형 HTML 슬라이드와, 발표자가 그대로 사용할 수 있는 대본 문서 `posts/csts/csts_script.md`를 만든다.

**Context**

- 저장소는 빌드 없이 브라우저에서 바로 여는 정적 HTML 프로젝트다.
- 사용자 요청 조건:
  - `csts.md` 내용은 전부 포함
  - `csts.html` 생성
  - `csts_script.md` 생성
  - 최소 40페이지 이상
  - 50분 발표 분량
  - 16:9 슬라이드
  - 좌우 화살표로만 이동
  - 진행바, 버튼, 페이지 컨트롤 없음
  - 필요 시 `posts/csts/png/*.png` 이미지 사용

## Deliverables

1. `posts/csts/csts.html`
2. `posts/csts/csts_script.md`

## Design Direction

선택된 방향은 `균형형 강의형`이다.

- 원문 충실도를 유지하되, 긴 표와 실습 파트를 여러 장으로 나눠 가독성을 확보한다.
- 슬라이드는 발표용 흐름 중심으로 재구성하지만, 원문의 모든 정의, 분류, 표, 실습, 코드, 이미지 참조를 포함한다.
- 화면은 한 장당 하나의 메시지가 보이도록 유지하고, 긴 표는 분할하거나 비교 카드로 재배치한다.
- 대본은 슬라이드 번호별 1분 내외 설명을 기본으로 하되, 실습 파트는 1.5분까지 확장 가능하게 작성한다.

## Slide Count Target

- 목표 분량: `44~48장`
- 발표 시간: `약 50분`
- 운영 방식:
  - 일반 개념 슬라이드: 40~70초
  - 표/비교 슬라이드: 50~80초
  - 실습/예시 슬라이드: 70~110초

## Visual / Interaction Rules

### Layout

- 16:9 고정 프레젠테이션 레이아웃
- 단일 HTML 문서 내 여러 `section.slide`
- 한 번에 한 장만 보이는 구조
- 텍스트가 많은 표는 2열 카드나 분할 표 레이아웃 사용

### Navigation

- `ArrowLeft`, `ArrowRight`만 허용
- 버튼 없음
- 진행바 없음
- 화면 하단 컨트롤 없음
- 클릭 이동 없음
- 페이지 번호는 있어도 되지만, 진행바 형태는 금지

### Tone

- 교육용/세미나용 강의 슬라이드
- 과도한 장식보다 구조와 가독성 우선
- 기존 저장소의 독립형 HTML 발표 문서 패턴을 따른다

## Script Design

- 파일: `posts/csts/csts_script.md`
- 형식:
  - 슬라이드 번호
  - 슬라이드 제목
  - 발표 대본
- 원칙:
  - 슬라이드 내용 반복이 아니라 설명형 문장으로 작성
  - 표는 비교 포인트를 말로 풀어준다
  - 실습은 “왜 이런 테스트를 만드는가”까지 설명한다
  - 총 분량은 50분 발표 기준으로 맞춘다

## Implementation Notes

- HTML/CSS/JS만 사용
- 외부 빌드 도구 없음
- 이미지 경로는 `./png/image1.png` 형식으로 사용
- 키보드 이벤트는 좌우 화살표만 처리
- 브라우저에서 파일을 직접 열어도 동작해야 한다
- 인쇄 친화성보다 화면 발표 경험을 우선한다

## Risks and Mitigations

1. 원문 누락 위험
   - 섹션별 체크리스트로 `csts.md` 전체를 대조한다.
2. 슬라이드 과밀 위험
   - 긴 표는 분할하고, 실습은 단계별로 나눈다.
3. 50분 분량 부족 또는 초과 위험
   - 슬라이드 수를 44장 이상으로 유지하고, 대본에서 각 장의 예상 시간을 고르게 배분한다.
4. 인터랙션 요구사항 위반 위험
   - 버튼, 진행바, 클릭 네비게이션을 넣지 않고 키보드 좌우 이동만 구현한다.

## Self-Review

- `csts.md` 주요 섹션과 세부 실습 항목이 모두 포함되도록 범위를 명시했다.
- 사용자 제약조건인 `40장 이상`, `50분`, `16:9`, `좌우 화살표만`, `버튼/진행바 없음`, `png 사용 가능`, `대본 파일 생성`을 모두 반영했다.
- 구현 단계에서 확인해야 할 위험 요소와 분할 원칙을 기록했다.
