02 · 시스템 개요

시스템 전체 블록도

중앙 제어 기반 구조

03 · 부팅 시퀀스

전원 유입부터 기능 활성화까지

VDD_EXT → 전원 → MCU → Flash → 기능 블록

이 보드의 핵심은 전원 품질이 곧 전체 동작의 출발점이라는 점이다. 전원 생성, 클럭/리셋 준비, MCU 부팅, Flash 접근, 기능 활성화가 연속적으로 이어진다.

STEP 1입력 전원 유입VDD_EXT가 시스템 시작점이 된다.
STEP 25V / 3.3V 생성레일이 분리되어 각 블록에 공급된다.
STEP 3클럭 / 리셋 준비부팅 기준 신호가 안정화되어야 한다.
STEP 4MCU 부팅MAX32560이 초기 제어를 시작한다.
STEP 5Flash 로드 후 기능 활성화통신과 표시 블록이 정상 상태로 전이된다.
앞 단계가 불안정하면 뒤 단계는 겉보기에는 살아 있어도 재현성 없는 실패를 만들 수 있다. 그래서 전원과 기준 신호를 먼저 확인하는 순서가 중요하다.

04 · 제어 중심부

핵심 제어부: MAX32560

Clock · Reset · JTAG · VBAT
  • MAX32560은 보드 전체 동작을 묶는 중앙 제어부다.
  • 12MHz 클럭은 일반 동작의 기준, 32.768kHz는 저전력 기준을 제공한다.
  • 리셋 품질은 부팅 재현성과 직결된다.
  • JTAG / USB는 디버그와 초기 검증 경로로 연결된다.
  • VBAT 연계는 전원 상실 시 상태 유지 전략과 연결된다.
MCU 검토의 핵심은 성능 수치보다 정상 시작 조건이다. 클럭, 리셋, 전원 분리, 디버그 접근성은 모두 같은 조건 묶음으로 해석해야 한다.
12MHz X-TAL

일반 동작의 기준 주파수. MCU 기동과 주요 처리 흐름의 타이밍 기준.

32.768kHz X-TAL

저전력 모드와 장기 유지 상태에서 기준 역할.

MCU_RSTN

불안정한 리셋은 간헐 부팅 실패와 디버깅 난이도를 높인다.

JTAG / USB_DP / USB_DM

디버그, 펌웨어 확인, 초기 bring-up 경로에 연결된다.

05 · 저장과 보안

외부 저장장치와 Secure 연동

W25Q128JVPIM · QSPI
  • W25Q128JVPIM은 고속 외부 Flash로서 코드, 설정, 데이터를 저장한다.
  • QSPI 연결은 부팅 초기 접근성과 처리 속도를 동시에 요구한다.
  • Flash는 단순 저장장치가 아니라 Secure Box 정책과도 연동되는 구성요소다.
  • 즉, 저장 구조와 보안 구조를 분리해서 보면 실제 동작 맥락을 놓치게 된다.
Flash 연동은 단순 저장 속도보다 부팅 경로에 얼마나 깊게 연결되어 있는지, 그리고 보안 영역과 어떤 관계를 갖는지가 더 중요한 해석 포인트다.

06 · 물리 보안

물리 보안 구조

Tamper · Mesh · Layer
  • Tamper Switch는 개봉과 구조 변경의 시작점을 감지한다.
  • Mesh Pattern은 절단과 우회 시도를 탐지하는 핵심 구조다.
  • 다층 배치는 단순 점퍼 우회 난도를 높인다.
  • 보안 설계의 목적은 단순 차폐가 아니라 침입 탐지와 차단이다.
물리 보안은 tamper 경로가 실제 감지 회로로 닫히는지, 우회 가능한 단순 경로가 남아 있지 않은지, 레이어 설명과 신호 쌍이 일치하는지를 중심으로 본다.

07 · NFC 경로

NFC 통신 블록

PN5180 · Antenna · Matching
  • PN5180이 NFC 제어 칩이며, MCU와는 SPI로 연결된다.
  • 안테나는 실제 통신 성능을 결정하는 최종 물리 요소다.
  • RF Matching은 칩과 안테나 사이의 효율과 손실을 조정한다.
  • EMC Filter는 잡음 억제와 규격 대응 측면에서 중요하다.
  • RF_PWR_ON은 동작 제어 신호이며 polarity 해석이 중요하다.
NFC 성능은 칩 단품보다 주변 패시브와 안테나 구조에 크게 좌우된다. 따라서 RF 경로를 한 줄의 체인으로 이해해야 실제 검토가 가능하다.

08 · NFC 검토

NFC 설계 검토 포인트

Polarity · Matching · EMC

RF_PWR_ON 해석

원문에는 LOW = RF POWER ON, HIGH = RF POWER OFF라는 주의가 있다. bring-up 시 polarity를 반대로 이해하면 칩은 살아 있어도 RF 동작이 죽어 있을 수 있다.

안테나와 매칭의 상호 영향

칩 설정만으로 성능을 판단하면 안 된다. 안테나 패턴, 매칭 부품값, 주변 접지 구조가 함께 결과를 만든다.

EMC 민감 구간

RF 경로 주변 전원/디지털 노이즈가 들어오면 성능 편차가 커질 수 있다. GND 구조와 필터 배치 해석이 필요하다.

디버깅 우선순위

제어 신호 확인 → 전원 상태 확인 → RF enable 상태 확인 → 매칭과 안테나 경로 확인 순서로 보는 것이 효율적이다.

이 장은 부품 목록을 외우기 위한 슬라이드가 아니라 검토 질문을 정리하기 위한 슬라이드다. 핵심은 칩 자체보다 RF chain 전체가 정상 조건을 만족하는지다.

09 · 외부 인터페이스

UART/USB 외부 인터페이스

Debug · Host Connection
  • UART0 / UART1은 디버그와 단순 통신 확인에 적합한 경로다.
  • USB_DP / USB_DM은 외부 호스트와 연결되는 차동 신호 경로다.
  • USB는 배선 품질과 차동 임피던스 해석이 중요하다.
  • ESD 보호는 외부 인터페이스 안정성에 직접 연결된다.
  • UART는 내부 확인용, USB는 외부 연결용이라는 역할 구분이 필요하다.
외부 인터페이스는 핀 이름보다 외부 리스크가 어떤 경로로 시스템 안으로 들어오는지를 보여주는 것이 중요하다.

10 · 상태 표시

LED 상태 표시 블록

LP5012 · RGB LED
  • LP5012가 다채널 LED 드라이버 역할을 수행한다.
  • RGB LED는 전원, 통신, 오류 상태를 직관적으로 표현한다.
  • PWM 기반 밝기 제어는 단순 on/off보다 더 많은 상태 표현을 가능하게 한다.
  • 기술 검토 관점에서는 사용자 표시 수단이면서 동시에 진단 인터페이스다.
상태 표시 블록은 부가 기능처럼 보이지만 bring-up과 현장 진단에서 빠른 판단 근거가 되기 때문에 실제 운용 관점에서는 비중이 크다.

11 · 메인 전원

메인 전원 생성부

VDD_EXT · 5V · 3.3V · PWR_GOOD
  • VDD_EXT가 시스템 입력 전원이다.
  • TPS61252DSG를 통해 5V 레일이 형성되고, 추가 전원 구조를 통해 3.3V가 공급된다.
  • PWR_GOOD은 레일이 정상 범위에 들어왔는지 알리는 기준 신호다.
  • 레일 분리는 노이즈와 오동작을 줄이는 데 중요하다.
전원 구조는 수치보다 순서가 중요하다. 어느 레일이 먼저 살아야 하는지, 어떤 신호가 정상 상태를 증명하는지가 핵심 설명 축이다.

12 · 백업 전원

CR2032 백업 전원

VBAT · PFO · RESET
  • CR2032는 전원 상실 시 유지 동작을 위한 백업 에너지원이다.
  • TPS3619-33DGK가 전압 감시와 전환 조건 판단을 담당한다.
  • VBAT, PFO, RESET은 저전력 전환과 상태 보존 해석의 핵심 신호다.
  • 유지 전류 가정과 온도 조건은 실제 유지 시간을 크게 바꾼다.
백업 전원은 단순 연결 설명보다 계산 가정이 맞는지를 함께 검토해야 한다. 그래서 전류 가정과 유지 시간 해석이 같이 들어간다.

13 · 보드 규칙

설계 규칙과 레이아웃 원칙

표지 메모 → 검토 규칙

배치 검토 우선

원문 cover note에 따라 배치 검토 이후 진행한다는 전제가 있다. 즉, 부품 선택 이후에도 배치 타당성 검토가 남아 있다.

BYPASS 배치

용량이 작은 bypass capacitor를 IC 가까이에 두는 원칙은 노이즈와 순간 응답의 기본 규칙이다.

전원과 내층 연결

모든 전원은 처리 후 내층 연결이라는 메모는 레일 정리와 리턴 경로 설계를 의식한 지침으로 읽어야 한다.

이름과 핀 표기

BOARD COPPER, TP, CONNECTOR 등 명칭 기준을 통일해야 검토와 생산 문서 간 혼선을 줄일 수 있다.

적층 기준

무연 FR-4 / HAL / 4-LAYER / 1.2T / GREEN이라는 cover 정보는 제조 조건과 구조 가정을 제공한다.

설명 가능한 규칙

이 규칙들은 체크리스트가 아니라 왜 필요한지 설명 가능해야 실제 리뷰에서 의미가 있다.

표지 메모는 단순 참고사항이 아니라 후속 블록 설명 전반에 공통으로 적용되는 규칙이다. 이 장은 그 규칙을 한 번에 정리하는 역할을 한다.

16 · 핵심 정리

핵심 정리

Five System Axes
제어

MAX32560이 부팅과 블록 연결의 중심이다.

저장 / 보안

Flash와 Secure Box는 부팅과 정책 적용을 함께 지배한다.

외부 통신

NFC, UART, USB는 각각 다른 리스크와 역할을 가진다.

상태 표시

LED는 사용자 표시이면서 현장 진단 도구다.

안정성

전원과 배터리 구조가 전체 재현성을 결정한다.

TNP20 HW는 단일 기능 보드가 아니라, 보안과 기능 블록이 한 장의 PCB 위에서 상호의존적으로 동작하는 시스템 보드다. 따라서 검토도 블록별 확인을 넘어서 공통 기반과 전파 경로를 함께 봐야 한다.

17 · 용어 정리

핵심 단어와 개념 정리

검토 용어 기준
MCU

주변 장치를 통합 제어하는 중심 프로세서. 이 보드에서는 MAX32560이 해당한다.

QSPI

고속 외부 Flash 연결에 사용하는 직렬 인터페이스. 부팅 경로의 성능과 안정성에 영향이 있다.

Tamper

물리 침입, 개봉, 절단 시도를 감지하는 구조 또는 신호 체계.

Secure Box

민감 회로나 보안 감지 구조를 보호하고 연동하는 개념적 보안 영역.

PWM

LED 밝기 제어처럼 연속적인 상태 표현을 가능하게 하는 펄스 폭 제어 방식.

PWR_GOOD

전원 레일이 정상 범위에 들어왔음을 알려주는 상태 신호.

PFO

전원 이상 또는 저전압 조건을 알려주는 모니터 출력 신호.

ESD

정전기 방전에 대한 보호 설계. 외부 인터페이스 안정성과 직접 연결된다.

RF

무선 주파수 경로. NFC 성능은 RF chain 전체가 결정한다.

EMC

전자파 간섭 억제와 적합성 관점. RF 경로와 주변 노이즈 관리에 중요하다.
